# 舆情分析 Skill

## 功能概述

用于对抖音、小红书等平台的内容和评论进行舆情分析，包含数据清洗、定性定量分析、报告生成三个步骤。

## 使用场景

当用户提供原始数据（抖音/小红书评论或内容）时，使用本skill进行分析。

## 工作流程

### 第一步：数据清洗
输入：内容和评论两个原始文件
输出：清洗后的Excel（第一个sheet为内容，第二个sheet为评论）
命令：`python scripts/clean_data.py <内容文件> <评论文件> <输出文件> --merge`

**内容数据保留列**：内容、标题、发布时间、点赞数、评论数、作者等
**评论数据保留列（7列）**：
- 评论id (comment_id)
- 评论时间 (create_time) - Unix时间戳转换为"2026-05-04 19:49:10"格式
- ip_location
- 评论内容 (content)
- user_id
- short_user_id
- 昵称 (nickname)
- 二级评论数 (sub_comment)
- 评论获赞 (like_count)

### 第二步：分析
输入：清洗后的数据
输出：分析结果（情感分布、品牌/产品提及、痛点词统计）
命令：`python scripts/analyze.py <清洗后数据> [输出文件]`

### 第三步：报告生成
输入：分析结果
输出：Word和Excel两种格式的报告（同时生成）
命令：`python scripts/report.py <分析结果> <输出报告> [格式]`
默认格式：both（Excel + Word同时生成）

## 分析能力

- **情感分析**：正面/负面/中性
- **品牌提及**：识别a2及其竞品品牌
- **产品提及**：识别具体产品型号
- **痛点分析**：按5个阶段分类统计（孕期、1段、2段、3段、喂养痛点）
- **品牌纠错**：自动纠正常见错误写法

## 分析逻辑文档

位于 `analysis_logic/` 目录：
- `vocabulary.md` - 痛点关键词
- `brands_products.md` - 品牌产品信息
- `analysis_rules.md` - 识别规则和纠错表

## 关键词汇

**a2品牌产品**：a2至初、a2紫白金、a2紫曜

**竞品品牌**：爱他美、美素佳儿、金领冠、飞鹤、君乐宝等14个

**痛点阶段**：孕期、1段(0-7月)、2段(7-12月)、3段(1岁+)、喂养痛点

## 调用示例

用户说"分析这份数据"时：
1. 先读取 `analysis_logic/` 下的最新文档
2. 执行数据清洗
3. 执行分析
4. 生成报告
